# liveplots

Real-time live plot server using XML-RPC and Gnuplot.

## Overview

`liveplots` serves a minimalistic plotting API through an XML-RPC server.
Plots are generated by [Gnuplot](http://www.gnuplot.info/).

The purpose of this package is to monitor long-running computations without
interfering with them (i.e. slowing them down).

Multiple plot servers can be started, each on its own daemon process.

## Features

- :material-chart-line: **Real-time plotting** via XML-RPC + Gnuplot
- :material-file-eye: **Cross-platform file monitoring** via [watchdog](https://python-watchdog.readthedocs.io/)
- :material-thread: **Non-blocking** — plot commands are queued and processed asynchronously
- :material-console: **Multiple plot types**: scatter, lines, histograms, multiplot layouts

## Requirements

- Python 3.12+
- Gnuplot (install via `apt install gnuplot` on Debian/Ubuntu, `brew install gnuplot` on macOS)

## Installation

```bash
uv add liveplots
```

Or with pip:

```bash
pip install liveplots
```

## Quick Start

```python
from numpy.random import normal
from liveplots import PlotServer

pserver = PlotServer(port=0, persist=1)

data = normal(0, 1, 1000).tolist()
data2 = normal(4, 1, 1000).tolist()

pserver.lines([data, data2], [], ["data", "data2"], "Two plots")
pserver.flush_queue()
```

More examples in the [`examples/`](https://github.com/fccoelho/liveplots/tree/master/examples) directory.
